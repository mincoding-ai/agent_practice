# Commit Convention

## 커밋 메시지 구조

```
<type> subject

body

footer
```

---

## Type 종류

| Type | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 작성 및 수정 |
| `style` | 코드 포맷팅, 세미콜론 누락 등 (기능 변경 없음) |
| `refactor` | 코드 리팩토링 (기능 변경 없음) |
| `test` | 테스트 코드 추가 및 수정 |
| `chore` | 빌드 설정, 패키지 업데이트 등 기타 작업 |

---

## 규칙

1. **header(헤더)** 는 필수이며 50자 이내로 작성한다.
2. **type** 은 `< >` 문자로 감싸서 작성한다. 예: `<feat>`, `<fix>`
3. **커밋 메시지(subject, body)** 는 한글로 작성한다.
4. **subject** 는 명령형으로 작성하며 마침표를 붙이지 않는다.
5. **body** 는 선택 사항이며 무엇을, 왜 변경했는지 설명한다.
6. **footer** 는 선택 사항이며 이슈 번호 등을 참조할 때 사용한다.

---

## 예시

```
<feat> 로그인 기능 추가

이메일과 비밀번호를 이용한 로그인 기능을 구현함.
JWT 토큰 발급 및 세션 관리 포함.

Closes #42
```

```
<fix> 긴급 Fix
```

```
<fix> 회원가입 시 이메일 중복 체크 오류 수정
```

```
<docs> README에 설치 방법 추가
```
